# Requirements: Repository Description

## User Stories

**As a** developer discovering the kodit repository  
**I want** a short, clear description of what the project does  
**So that** I can quickly understand its purpose without reading extensive documentation

## Acceptance Criteria

- [ ] A markdown file exists with a brief description of the kodit repository
- [ ] Description is concise (1-3 sentences)
- [ ] Description captures the core purpose: MCP server for code indexing that connects AI coding assistants to external codebases
- [ ] File is placed in an appropriate location (e.g., root or docs folder)
